# [자료구조] 해시

## 한 줄 요약

> <u>`해시`는 `키`를 `해시 함수`로 바꿔 빠르게 찾고 저장하는 자료구조다.</u>

해시는 키를 바로 인덱스로 쓰지 않고, 해시 함수를 통해 고정된 위치로 바꿔 관리한다.

핵심 키워드: `키`, `해시 함수`, `해시 테이블`, `충돌`, `rehash`

<br />

## 1. 핵심 특징

> <u>해시는 평균적으로 빠르지만, 충돌이 많아지면 느려질 수 있다.</u>

- 저장과 탐색의 평균 시간 복잡도는 O(1)에 가깝다.
- 키를 해시 값으로 변환해서 저장한다.
- 충돌이 발생하면 별도 처리 방식이 필요하다.
- 데이터가 많아지면 재해시(rehash)가 일어날 수 있다.

해시는 빠른 검색, 빈도수 카운트, 중복 검사에 자주 쓴다.

## 2. 충돌

> <u>`충돌`은 서로 다른 키가 같은 위치로 가는 상황이다.</u>

- 해시 함수가 아무리 좋아도 충돌은 완전히 피하기 어렵다.
- 충돌이 많아지면 성능이 떨어진다.
- 대표 처리 방식은 `체이닝`과 `오픈 어드레싱`이다.

### 체이닝

- 같은 버킷에 연결 리스트나 다른 구조로 여러 값을 저장한다.
- 구현이 비교적 단순하다.

### 오픈 어드레싱

- 빈 칸을 찾으며 다른 위치에 저장한다.
- 연속된 탐색이 길어질 수 있다.

## 3. 기본 코드

```java
import java.util.HashMap;
import java.util.HashSet;
import java.util.Map;
import java.util.Set;

public class HashExample {
    public static void main(String[] args) {
        Map<String, Integer> count = new HashMap<>();
        count.put("apple", count.getOrDefault("apple", 0) + 1);
        count.put("banana", count.getOrDefault("banana", 0) + 1);
        count.put("apple", count.getOrDefault("apple", 0) + 1);

        Set<String> seen = new HashSet<>();
        seen.add("A");
        seen.add("B");
        seen.add("A");

        System.out.println(count.get("apple"));
        System.out.println(seen.contains("B"));
    }
}
```

## 4. 자주 나오는 포인트

### 1) 해시와 해시 테이블

해시는 값을 빠르게 찾기 위한 아이디어이고, 해시 테이블은 그 아이디어를 구현한 구조다.

### 2) 자주 쓰는 문제

- 빈도수 세기
- 중복 확인
- 두 수의 합
- 문자열 그룹화
- 캐시 구현

### 3) 실전 함정

| 함정 | 설명 |
| --- | --- |
| 해시 충돌 | 같은 값처럼 보이는 서로 다른 키가 생길 수 있다. |
| rehash | 테이블이 커지면 다시 배치하면서 비용이 들 수 있다. |
| key equality | `hashCode`와 `equals`가 맞아야 한다. |
| 순서 보장 | `HashMap`은 순서를 보장하지 않는다. |
| null 처리 | 언어별로 `null` 허용 여부가 다르다. |

### 4) Java에서 볼 것

- `HashMap`
- `HashSet`
- `LinkedHashMap`
- `TreeMap`
