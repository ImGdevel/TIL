# 스택 / 큐

> 태그: `#ds` `#stack` `#queue`<br>
> 작성일: 2026-06-25<br>
> 최종 수정일: 2026-06-25

## 정의

`스택`은 마지막에 넣은 원소를 먼저 꺼내는 `LIFO` 구조이고, `큐`는 먼저 들어온 원소를 먼저 꺼내는 `FIFO` 구조다.

## 특징 / 상세

핵심 키워드: `LIFO`, `FIFO`, `push`, `pop`, `top`, `front`

스택은 마지막에 넣은 것을 먼저 꺼내고, 큐는 먼저 들어온 것을 먼저 꺼낸다.

### 1. Stack

> 스택은 마지막에 넣은 원소를 먼저 꺼내는 구조다.

- `push`: 넣기
- `pop`: 꺼내기
- `top`: 맨 위 확인

스택은 깊이 우선 처리, 되돌리기, 함수 호출 관리에서 자주 쓴다.

#### 장점

- 구현이 단순하다.
- 뒤로 가기, 되돌리기와 잘 맞는다.
- 최근 입력을 먼저 처리하기 좋다.

#### 단점

- 중간 원소 접근이 불편하다.
- 순서가 뒤집히는 구조에는 맞지 않는다.

### 2. Queue

> 큐는 먼저 들어온 원소를 먼저 꺼내는 구조다.

- `push` / `enqueue`: 넣기
- `pop` / `dequeue`: 꺼내기
- `front`: 맨 앞 확인

큐는 대기열, 작업 처리, 너비 우선 탐색에서 자주 쓴다.

#### 장점

- 도착 순서를 보존한다.
- 작업 대기열에 잘 맞는다.
- 순차 처리 흐름이 자연스럽다.

#### 단점

- 중간 삽입/삭제는 보통 다루지 않는다.
- 뒤에서 넣고 앞에서 빼는 흐름에 맞춰야 한다.

### 3. 비교

| 연산 | 스택 | 큐 |
| --- | --- | --- |
| 넣기 | `push` | `push` / `enqueue` |
| 빼기 | `pop` | `pop` / `dequeue` |
| 확인 | `top` | `front` |
| 처리 순서 | `LIFO` | `FIFO` |
| 대표 사용처 | 재귀, 되돌리기, DFS | 대기열, 버퍼, BFS |
| 접근 방식 | 뒤에서 넣고 뒤에서 뺀다 | 뒤에서 넣고 앞에서 뺀다 |

> 스택은 최근 항목 우선, 큐는 도착 순서 우선이라고 보면 된다.

### 4. 핵심 키워드

- `Stack`: `LIFO`
- `Queue`: `FIFO`
- `push` / `pop` / `top` / `front`
- `괄호 검사`
- `DFS stack`
- `BFS queue`
- `monotonic stack`
- `circular queue`
- `overflow` / `underflow`

### 5. 기본 코드

```java
import java.util.ArrayDeque;
import java.util.Deque;
import java.util.Queue;

public class StackQueueExample {
    public static void main(String[] args) {
        Deque<Integer> stack = new ArrayDeque<>();
        Queue<Integer> queue = new ArrayDeque<>();

        stack.push(1);
        stack.push(2);
        stack.push(3);

        System.out.println(stack.peek());
        stack.pop();

        queue.offer(1);
        queue.offer(2);
        queue.offer(3);

        System.out.println(queue.peek());
        queue.poll();
    }
}
```

### 6. 자주 나오는 포인트

#### 1) 스택과 큐의 차이

스택은 `LIFO`, 큐는 `FIFO`다.

#### 2) 어디에 쓰이나

- 스택: 괄호 검사, 함수 호출, 재귀, 뒤로 가기, `DFS`
- 큐: 작업 대기열, 버퍼, 레벨 순회, `BFS`

#### 3) 구현할 때 주의점

> 스택은 `top`, 큐는 `front`를 기준으로 생각해야 헷갈리지 않는다.

#### 4) 실전 함정

| 함정 | 설명 |
| --- | --- |
| 괄호 검사 | `")("`, `"(()"`, `""` 같은 입력을 먼저 확인해야 한다. |
| queue 구현 | `front/rear` wrap-around를 놓치기 쉽다. |
| stack pop | 비어 있을 때 `pop`하면 안 된다. |
| BFS | `visited`를 enqueue 시점에 찍어야 중복을 막는다. |
| DFS | recursion stack overflow 가능성이 있다. |
| monotonic stack | 단조성이 깨지면 답이 틀어진다. |
| circular queue | 원형 인덱스 갱신이 핵심이다. |
