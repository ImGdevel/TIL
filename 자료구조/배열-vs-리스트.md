# 배열 vs 리스트

> 태그: `#ds` `#array` `#linked-list`<br>
> 작성일: 2026-06-25<br>
> 최종 수정일: 2026-06-25

## 정의

`배열`은 연속된 메모리에 원소를 저장해 인덱스 접근이 빠른 자료구조이고, `리스트`(연결 리스트)는 노드를 포인터로 연결해 삽입/삭제는 빠르지만 임의 접근은 느린 자료구조다.

## 특징 / 상세

핵심 키워드: `연속 메모리`, `인덱스 접근`, `캐시 효율`, `삽입/삭제`

배열은 인덱스 접근과 캐시 효율이 좋아 읽기가 빠르지만 중간 삽입과 삭제가 비싸다. 연결 리스트는 노드를 알고 있을 때 수정이 빠르지만 임의 접근과 실전 성능에서 불리한 경우가 많다.

여기서 리스트는 보통 연결 리스트를 기준으로 보거나, 언어에 따라 동적 배열 기반 List를 가리킬 수 있다.

비교할 때는 저장 방식, 접근 속도, 삽입/삭제, 메모리, 실무 사용처를 보면 된다.

### 1. 배열

배열은 메모리상에 연속적으로 저장된다.

> 핵심은 인덱스 접근이다.

- arr[i] 접근이 빠르다.
- 크기가 고정되면 관리가 단순하다.
- 연속 공간이 필요해서 중간 삽입/삭제는 비싸다.

#### 장점

- 임의 접근이 O(1)이다.
- 캐시 효율이 좋다.
- 구현이 단순하다.

#### 단점

- 크기 변경이 불편하다.
- 중간 삽입/삭제 시 뒤를 밀어야 한다.
- 연속 공간이 필요하다.

### 2. 리스트

리스트는 상황에 따라 의미가 다르다.

> 보통 자료구조에서는 연결 리스트를 뜻하는 경우가 많다.

연결 리스트는 노드가 포인터로 이어진 구조다.

- 삽입/삭제가 빠를 수 있다.
- 대신 임의 접근은 느리다.
- 다음 노드를 따라가야 한다.

#### 장점

- 중간 삽입/삭제가 쉽다.
- 크기 변경이 자연스럽다.
- 앞뒤로 이어 붙이기 좋다.

#### 단점

- arr[i] 같은 직접 접근이 느리다.
- 포인터 저장 때문에 메모리를 더 쓸 수 있다.
- 캐시 효율이 낮다.

### 3. 비교

| 연산 | 배열 | 연결 리스트 |
| --- | --- | --- |
| 인덱스 접근 | O(1) | O(N) |
| 값 탐색 | O(N) | O(N) |
| 중간 삽입 | O(N) | 위치를 알고 있으면 O(1) |
| 중간 삭제 | O(N) | 노드를 알고 있으면 O(1) |
| 앞쪽 삽입/삭제 | O(N) | O(1)에 가깝다 |
| 뒤쪽 삽입/삭제 | O(1)~O(N)<br />동적 배열은 amortized O(1) | O(1)에 가깝다 |
| 메모리 지역성 | 좋음 | 나쁨 |
| 추가 메모리 | 적음 | 포인터 필요 |
| 캐시 효율 | 좋음 | 나쁨 |
| 크기 변경 | 동적 배열은 유연하다 | 유연하다 |
| 임의 원소 삭제 | 뒤 원소를 당겨야 한다 | 노드만 알면 연결만 수정한다 |
| 구현 난이도 | 낮다 | 상대적으로 높다 |

> 그래서 조회가 많으면 배열, 삽입/삭제가 많으면 리스트를 생각한다.

### 4. 자주 나오는 포인트

#### 1) 배열이 더 빠른가?

대부분의 읽기는 배열이 더 유리하다.

#### 2) 리스트가 더 좋은가?

삽입/삭제가 많다고 해서 무조건 리스트가 좋은 건 아니다.

> 실제로는 어디에 삽입/삭제가 자주 일어나는지, 탐색이 얼마나 많은지를 같이 봐야 한다.

#### 3) 실무에서 자주 보는 기준

- 탐색 위주면 배열
- 순차 처리 위주면 배열
- 중간 연결/스플라이싱이 많으면 리스트

#### 4) 언어별 주의

- Java List는 보통 인터페이스다.
- ArrayList는 동적 배열이다.
- LinkedList는 연결 리스트다.

### 5. 엣지 케이스 / 함정

| 함정 | 설명 |
| --- | --- |
| 리스트 삽입은 무조건 O(1) | 위치를 이미 알고 있을 때만 성립한다. |
| 배열 탐색은 O(1) | 인덱스 접근은 O(1)이지만, 값 탐색은 O(N)이다. |
| 동적 배열 push_back | 평균 O(1)이지만, resize 시 O(N)이 될 수 있다. |
| linked list 메모리 | 포인터 때문에 overhead가 있다. |
| CPU cache | 배열이 실전 성능에서 유리한 경우가 많다. |
| vector erase | iterator 이후 원소가 이동해서 O(N)이다. |
| list random access | 불가하거나 O(N)이다. |
