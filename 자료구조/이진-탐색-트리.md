# 이진 탐색 트리

## 한 줄 요약

> <u>BST는 왼쪽 < 부모 < 오른쪽 규칙으로 값을 빠르게 찾기 위한 이진 트리다.</u>

이진 탐색 트리는 왼쪽 자식 < 부모 < 오른쪽 자식 규칙을 만족하는 이진 트리다.

검색, 삽입, 삭제를 빠르게 하기 위한 자료구조다. 평균적으로 O(log N)에 가깝게 동작하지만, 한쪽으로 치우치면 O(N)이 된다.

## 1. 핵심 특징

> <u>중위 순회 결과가 오름차순이라서 정렬된 결과처럼 쓸 수 있다.</u>

- 왼쪽 서브트리의 값은 현재 노드보다 작다.
- 오른쪽 서브트리의 값은 현재 노드보다 크다.
- 중위 순회 결과는 오름차순이다.

## 2. 기본 코드

### C++

```cpp
#include <bits/stdc++.h>
using namespace std;

struct Node {
    int data;
    Node* left;
    Node* right;

    Node(int x) : data(x), left(nullptr), right(nullptr) {}
};

Node* insert(Node* root, int x) {
    if (root == nullptr) return new Node(x);
    if (x < root->data) root->left = insert(root->left, x);
    else if (x > root->data) root->right = insert(root->right, x);
    return root;
}

bool search(Node* root, int x) {
    if (root == nullptr) return false;
    if (root->data == x) return true;
    if (x < root->data) return search(root->left, x);
    return search(root->right, x);
}

void inorder(Node* root) {
    if (root == nullptr) return;
    inorder(root->left);
    cout << root->data << ' ';
    inorder(root->right);
}
```

class Node:

        self.left = None
        self.right = None


def insert(root, x):
    if root is None:
        return Node(x)
    if x < root.data:
        root.left = insert(root.left, x)
    elif x > root.data:
        root.right = insert(root.right, x)
    return root

    if root is None:
        return False
    if root.data == x:
        return True
    if x < root.data:
        return search(root.left, x)

## 3. 빈출 문제

### 1) BST 검증

> <u>검증은 부모와 직접 비교하기보다 범위 제한 DFS가 더 안전하다.</u>

- 부모와 직접 비교
- 범위 제한을 두고 DFS

핵심:

- 왼쪽은 (-inf, root)
- 오른쪽은 (root, +inf)

### 2) 중위 순회

- 왜 정렬 결과가 나오는가
- 재귀 대신 반복으로도 가능한가

핵심:

- Left -> Root -> Right
- BST에서는 오름차순 출력

### 3) 삽입 / 삭제

> <u>삭제는 자식의 개수에 따라 0개 / 1개 / 2개로 나눠 처리한다.</u>

- 새 값을 어디에 넣는가
- 삭제 시 자식이 0개 / 1개 / 2개일 때 처리

핵심:

- 0개: 그냥 제거
- 1개: 자식으로 대체
- 2개: 보통 오른쪽 서브트리의 최솟값(또는 왼쪽 최댓값)으로 대체

### 4) K번째 작은 수

> <u>K번째 작은 수는 중위 순회와 잘 맞는다.</u>

- BST에서 k번째로 작은 값 찾기

핵심:

- 중위 순회를 하면 작은 값부터 나온다.
- 방문 횟수를 세면 된다.

## 4. 자주 나오는 포인트

> 평균 O(log N), 최악 O(N)이다.
> 편향 트리는 사실상 연결 리스트처럼 된다.
> 중복값 처리는 문제 조건을 먼저 확인해야 한다.
> 균형이 깨지면 AVL, Red-Black Tree 같은 균형 트리를 쓴다.
