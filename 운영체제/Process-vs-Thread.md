# [운영체제] Process vs Thread

## 한 줄 요약

> <u>`Process`는 자원과 주소 공간을 가진 실행 단위이고, `Thread`는 같은 프로세스 안에서 실행 흐름을 나눈 단위다.</u>

프로세스는 서로 독립적인 메모리 공간을 가지는 반면, 스레드는 같은 프로세스의 메모리를 공유해서 통신이 쉽지만 동기화 문제가 생긴다.

핵심 키워드: `address space`, `memory isolation`, `thread 공유`, `IPC`, `race condition`

<br />
<br />

---

## 1. Process

> <u>프로세스는 OS가 자원을 할당하는 실행 단위다.</u>

- 코드/데이터/힙/스택 같은 주소 공간을 가진다.
- 다른 프로세스와 메모리는 기본적으로 격리된다.
- 프로세스 간 통신은 보통 `IPC`를 쓴다.

<br />
<br />

---

## 2. Thread

> <u>스레드는 프로세스 내부의 실행 흐름 단위다.</u>

- 같은 프로세스의 힙/데이터 영역을 공유한다.
- 스레드마다 스택과 레지스터 같은 실행 문맥은 따로 가진다.
- 공유 메모리 때문에 `mutex`, `semaphore` 같은 동기화가 필요하다.

<br />
<br />

---

## 3. 비교

| 관점 | Process | Thread |
| --- | --- | --- |
| 메모리 | 격리 | 공유 |
| 통신 | IPC 필요 | 공유 메모리로 가능 |
| 생성/전환 비용 | 비교적 큼 | 비교적 작음 |
| 안전성 | 상대적으로 안전 | 동기화 문제 발생 |

<br />
<br />

---

## 4. 자주 나오는 포인트

- 스레드는 공유 자원 때문에 `race condition`이 생길 수 있다.
- 프로세스는 격리 때문에 안정성이 좋지만, 통신 비용이 든다.
