# [알고리즘/정렬] 머지 정렬

## 한 줄 요약

> <u>`머지 정렬`은 `분할 정복`으로 나누고 다시 `병합`해서 정렬하는 `안정 정렬`이다.</u>

배열을 반으로 나누고, 각각을 정렬한 뒤, 정렬된 두 배열을 합치면서 전체를 정렬한다.

핵심 키워드: `분할 정복`, `병합`, `안정 정렬`, `O(N log N)`, `추가 메모리`

## 1. 핵심 특징

> <u>머지 정렬은 항상 반씩 나눠서 처리하고, 마지막에 정렬된 결과를 합친다.</u>

- 분할 정복 알고리즘이다.
- 최악의 경우에도 O(N log N)이다.
- 안정 정렬이다.
- 보통 추가 배열이 필요하다.

머지 정렬은 정렬 기준이 같은 원소의 상대 순서를 유지해야 할 때 자주 쓴다.

## 2. 동작 방식

1. 배열을 반으로 나눈다.
2. 각각을 다시 머지 정렬한다.
3. 정렬된 두 구간을 합친다.

> <u>핵심은 이미 정렬된 두 구간을 빠르게 합치는 것이다.</u>

- 왼쪽 포인터와 오른쪽 포인터를 비교한다.
- 더 작은 값을 결과 배열에 넣는다.
- 한쪽이 끝나면 나머지를 그대로 붙인다.

## 3. 기본 코드

```java
import java.util.*;

public class MergeSortExample {
    static void mergeSort(int[] arr, int left, int right, int[] temp) {
        if (left >= right) return;

        int mid = (left + right) / 2;
        mergeSort(arr, left, mid, temp);
        mergeSort(arr, mid + 1, right, temp);
        merge(arr, left, mid, right, temp);
    }

    static void merge(int[] arr, int left, int mid, int right, int[] temp) {
        int i = left;
        int j = mid + 1;
        int k = left;

        while (i <= mid && j <= right) {
            if (arr[i] <= arr[j]) temp[k++] = arr[i++];
            else temp[k++] = arr[j++];
        }

        while (i <= mid) temp[k++] = arr[i++];
        while (j <= right) temp[k++] = arr[j++];

        for (int idx = left; idx <= right; idx++) {
            arr[idx] = temp[idx];
        }
    }

    public static void main(String[] args) {
        int[] arr = {5, 2, 4, 7, 1, 3, 2, 6};
        int[] temp = new int[arr.length];
        mergeSort(arr, 0, arr.length - 1, temp);
        System.out.println(Arrays.toString(arr));
    }
}
```

## 4. 자주 나오는 포인트

### 1) 왜 안정 정렬인가

같은 값이면 왼쪽 구간의 원소를 먼저 넣도록 만들 수 있기 때문이다.

### 2) 왜 O(N log N)인가

나누는 깊이가 log N이고, 각 단계에서 전체를 한 번씩 합치기 때문이다.

### 3) 실전 함정

| 함정 | 설명 |
| --- | --- |
| 배열 크기 | 병합용 임시 배열을 따로 잡아야 한다. |
| 인덱스 실수 | `left`, `mid`, `right` 경계를 헷갈리기 쉽다. |
| 안정성 | `<=`와 `<` 선택에 따라 안정 정렬이 깨질 수 있다. |
| 메모리 | 추가 배열이 필요해서 in-place가 아니다. |

### 4) 자주 같이 묻는 것

- 퀵 정렬과의 차이
- 안정 정렬 여부
- 추가 메모리 필요 여부
- 최악의 경우 시간 복잡도
