# [알고리즘/정렬] 퀵 정렬

## 한 줄 요약

> <u>`퀵 정렬`은 `pivot` 기준 `partition`으로 배열을 둘로 나누고, 각 구간을 재귀적으로 정렬하는 `분할 정복` 알고리즘이다.</u>

배열을 나눠서 정복하는 방식이라 평균 성능이 좋고, 구현도 비교적 단순하다.

핵심 키워드: `pivot`, `partition`, `평균 O(N log N)`, `최악 O(N^2)`, `in-place`, `unstable sort`, `recursion depth`, `Lomuto / Hoare`

<br />

## 1. 핵심 특징

> <u>퀵 정렬은 평균적으로 빠르지만, pivot 선택이 나쁘면 최악이 발생한다.</u>

- 평균 시간 복잡도는 O(N log N)이다.
- 최악의 경우 O(N^2)이다.
- 보통 `in-place`로 구현한다.
- 일반적으로 `unstable sort`다.
- 재귀 깊이는 평균적으로 log N이지만, 최악에는 O(N)까지 갈 수 있다.

최악은 분할이 계속 `0 : (N - 1)`처럼 한쪽으로 치우칠 때 생긴다. 예를 들어 이미 정렬된 배열에서 항상 끝 원소를 pivot으로 잡으면 치우치기 쉽다.

## 2. Partition (Lomuto 기준)

> <u>`partition`은 pivot을 기준으로 작은 값과 큰 값을 갈라서 pivot 위치를 확정한다.</u>

불변식:

- `[left, i)`는 `<= pivot`
- `[i, j)`는 `> pivot`
- `[j, right)`는 `unknown`
- `arr[right]`는 `pivot`

```java
static int partition(int[] arr, int left, int right) {
    int pivot = arr[right];
    int i = left;

    for (int j = left; j < right; j++) {
        if (arr[j] <= pivot) {
            swap(arr, i, j);
            i++;
        }
    }

    swap(arr, i, right);
    return i;
}

static void swap(int[] arr, int i, int j) {
    int temp = arr[i];
    arr[i] = arr[j];
    arr[j] = temp;
}
```

## 3. 기본 코드

```java
import java.util.Arrays;

public class QuickSortExample {
    static void quickSort(int[] arr, int left, int right) {
        if (left >= right) return;

        int pivot = partition(arr, left, right);
        quickSort(arr, left, pivot - 1);
        quickSort(arr, pivot + 1, right);
    }

    static int partition(int[] arr, int left, int right) {
        int pivot = arr[right];
        int i = left - 1;

        for (int j = left; j < right; j++) {
            if (arr[j] <= pivot) {
                i++;
                swap(arr, i, j);
            }
        }

        swap(arr, i + 1, right);
        return i + 1;
    }

    static void swap(int[] arr, int i, int j) {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }

    public static void main(String[] args) {
        int[] arr = {5, 2, 4, 7, 1, 3, 2, 6};
        quickSort(arr, 0, arr.length - 1);
        System.out.println(Arrays.toString(arr));
    }
}
```

## 4. 빈출 포인트 (참고)

### 1) 빈칸 채우기

`partition`에서 빈칸으로 가장 많이 묻는 건 `swap` 위치다.

```java
int partition(int[] arr, int left, int right) {
    int pivot = arr[right];
    int i = left;

    for (int j = left; j < right; j++) {
        if (arr[j] <= pivot) {
            // 빈칸
            i++;
        }
    }

    swap(arr, i, right);
    return i;
}
```

정답:

```java
swap(arr, i, j);
```

### 2) 시간 복잡도 설명

평균은 O(N log N), 최악은 O(N^2)이고, 최악은 분할이 한쪽으로 치우칠 때 생긴다.

## 5. 엣지 케이스

| 케이스 | 영향 |
| --- | --- |
| 이미 정렬된 배열 | `pivot` 선택이 나쁘면 최악 |
| 역정렬 배열 | 마찬가지로 최악 가능 |
| 모든 값이 같음 | `Lomuto`에서 분할 균형이 나쁠 수 있다 |
| 중복이 많음 | `3-way partition` 고려 |
| 재귀 깊이 | 최악 `O(N)` stack |
| stable 여부 | Quick Sort는 일반적으로 unstable |
| pivot 랜덤화 | 최악 확률을 완화한다 |

## 부록. 최악(O(N^2))이 되는 경우

> <u>매번 `partition`이 `0 : (N - 1)`처럼 한쪽으로만 갈라지면 `O(N^2)`가 된다.</u>

- 이미 정렬된 배열에서 `pivot = arr[right]` 같은 고정 전략을 쓰는 경우
- 역정렬 배열에서 `pivot = arr[left]` 같은 고정 전략을 쓰는 경우

- 배열: `[1, 2, 3, 4, 5]`, `pivot = arr[right]`
- 분할: `[1,2,3,4] | 5` -> `[1,2,3] | 4` -> ... (재귀 깊이 `O(N)`)

다음 전략으로 최악 가능성을 줄일 수 있다.

- `random pivot`: pivot을 무작위로 골라서 최악 입력에 대한 편향을 확률적으로 줄인다.
- `median-of-three`: `left`, `mid`, `right` 중 중앙값을 pivot으로 잡아 정렬된 입력에서 편향을 줄인다.
- `3-way partition`: `< pivot`, `== pivot`, `> pivot`으로 나눠 중복이 많은 입력에서 분할 균형을 개선한다.
