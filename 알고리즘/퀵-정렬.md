# [알고리즘/정렬] 퀵 정렬

## 한 줄 요약

> <u>`퀵 정렬`은 `pivot`을 기준으로 `partition`한 뒤, 각 구간을 재귀적으로 정렬하는 `분할 정복` 알고리즘이다.</u>

배열을 기준 값보다 작은 쪽과 큰 쪽으로 나누고, 나뉜 구간을 다시 정렬한다.

핵심 키워드: `pivot`, `partition`, `분할 정복`, `in-place`, `불안정 정렬`

<br />
<br />

---

## 1. 핵심 특징

> <u>퀵 정렬은 평균적으로 빠르고, 추가 메모리가 적은 편이다.</u>

- 평균 시간 복잡도는 O(N log N)이다.
- 최악의 경우 O(N^2)이다.
- 보통 in-place로 구현한다.
- 불안정 정렬이다.

퀵 정렬은 실제 구현에서 많이 쓰이지만, pivot 선택이 성능을 크게 좌우한다.

<br />

## 2. 동작 방식

1. pivot을 고른다.
2. pivot보다 작은 값은 왼쪽, 큰 값은 오른쪽으로 보낸다.
3. 왼쪽과 오른쪽 구간을 다시 퀵 정렬한다.

> <u>핵심은 partition 한 번으로 pivot의 위치를 확정하는 것이다.</u>

- partition 결과로 pivot은 정렬 위치에 놓인다.
- 왼쪽과 오른쪽은 각각 독립적으로 정렬한다.

## 3. 기본 코드

```java
import java.util.Arrays;

public class QuickSortExample {
    static void quickSort(int[] arr, int left, int right) {
        if (left >= right) return;

        int pivot = partition(arr, left, right);
        quickSort(arr, left, pivot - 1);
        quickSort(arr, pivot + 1, right);
    }

    static int partition(int[] arr, int left, int right) {
        int pivot = arr[right];
        int i = left - 1;

        for (int j = left; j < right; j++) {
            if (arr[j] <= pivot) {
                i++;
                swap(arr, i, j);
            }
        }

        swap(arr, i + 1, right);
        return i + 1;
    }

    static void swap(int[] arr, int i, int j) {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }

    public static void main(String[] args) {
        int[] arr = {5, 2, 4, 7, 1, 3, 2, 6};
        quickSort(arr, 0, arr.length - 1);
        System.out.println(Arrays.toString(arr));
    }
}
```

## 4. 자주 나오는 포인트

### 1) 왜 빠른가

구간을 잘 나누면 재귀 깊이가 낮아지고, 전체 비교 횟수가 줄어든다.

### 2) 왜 최악이 생기나

이미 정렬된 배열에서 나쁜 pivot을 고르면 한쪽으로 계속 쏠릴 수 있다.

### 3) 실전 함정

| 함정 | 설명 |
| --- | --- |
| pivot 선택 | 첫 원소만 고르면 편향이 심해질 수 있다. |
| 중복값 | `<=`와 `<` 처리에 따라 결과가 달라질 수 있다. |
| 재귀 깊이 | 최악의 경우 스택이 깊어질 수 있다. |
| 안정성 | 퀵 정렬은 보통 안정 정렬이 아니다. |

### 4) 자주 같이 묻는 것

- 머지 정렬과의 차이
- 안정 정렬 여부
- 평균과 최악의 시간 복잡도
- pivot 선택 방식
