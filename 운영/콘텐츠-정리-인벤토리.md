# 기존 콘텐츠 정리 인벤토리 v2

이 문서는 현재 `TIL` 원문과 `TIL.wiki` 정리본의 상태를 함께 관리하기 위한 운영 문서다.

원칙은 그대로 유지한다.

- `TIL`은 러프한 학습 저장소로 둔다.
- `wiki`는 읽기 좋은 대표 문서만 올린다.
- 원문은 삭제보다 유지와 병합을 우선한다.

## 1. 현재 상태 요약

- `TIL` Markdown 문서 수: `47`
- `TIL.wiki` Markdown 문서 수: `32`
- `wiki` 핵심 상태:
  - `Home.md` 구성 완료
  - `_Sidebar.md` 구성 완료
  - 비교안 2개 유지
  - 대표 주제 문서 다수 작성 완료

현재 단계는 “초기 설계”가 아니라 “1차 위키 승격 완료 후 운영 정리 단계”로 보는 편이 맞다.

## 2. 현재 운영 기준

문서를 판단할 때는 아래 다섯 가지 상태로 본다.

- `TIL 유지`: 원문, 초안, 학습 흔적, 러프한 메모
- `wiki 승격 완료`: 대표 문서가 이미 `wiki`에 존재
- `병합 소스 유지`: 여러 원문이 하나의 `wiki` 문서로 흡수된 상태
- `중복 정리 후보`: 비슷한 원문이 둘 이상 남아 있어 후속 정리가 필요한 상태
- `범위 외 문서`: 공부 위키보다 포트폴리오/프로젝트 성격이 강한 문서

## 3. 위키 작성 기준

현재 `wiki` 문체와 구조 기준은 아래로 고정한다.

- 구조는 “읽기 좋은 위키형”을 유지한다.
- 말투와 용어 선택은 Kafka 문서처럼 건조하고 설명형으로 맞춘다.
- 대주제 사이에는 `<br />` + `---` 패턴을 공통으로 사용한다.
- 처음 보는 사람도 흐름을 따라 이해할 수 있게 쓴다.
- 실무에서 자주 막히는 문제와 점검 방법을 함께 둔다.
