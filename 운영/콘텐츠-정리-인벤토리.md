# 기존 콘텐츠 정리 인벤토리 v5

이 문서는 현재 `TIL` 원문, `TIL.wiki` 기술 정리본, `Portfolio` 커리어/프로젝트 허브의 상태를 함께 관리하기 위한 운영 문서다.

원칙은 그대로 유지한다.

- `TIL`은 러프한 학습 저장소로 둔다.
- `TIL.wiki`는 읽기 좋은 대표 기술 문서만 올린다.
- `Portfolio`는 커리어, 프로젝트, 지원 준비 문서를 모아 둔다.
- 원문은 삭제보다 유지와 병합을 우선한다.

- `TIL` Markdown 문서 수: `48`

- `Portfolio` 핵심 상태:
  - `Home.md` 구성 완료

현재 단계는 “초기 설계”가 아니라 “1차 승격 완료 후 허브 분리 운영 단계”로 보는 편이 맞다.

## 2. 현재 운영 기준

문서를 판단할 때는 아래 여섯 가지 상태로 본다.

- `TIL 유지`: 원문, 초안, 학습 흔적, 러프한 메모
- `wiki 승격 완료`: 대표 기술 문서가 이미 `TIL.wiki`에 존재
- `병합 소스 유지`: 여러 원문이 하나의 `TIL.wiki` 문서로 흡수된 상태
- `중복 정리 후보`: 비슷한 원문이 둘 이상 남아 있어 후속 정리가 필요한 상태
- `Portfolio 승격 완료`: 커리어, 프로젝트, 지원 준비 문서가 별도 허브로 승격된 상태
- `raw 프로젝트 원문`: 프로젝트별 러프한 원문 메모로 남겨 두는 상태

## 3. 위키 작성 기준

현재 `TIL.wiki` 문체와 구조 기준은 아래로 고정한다.

- 구조는 “읽기 좋은 위키형”을 유지한다.
- 말투와 용어 선택은 Kafka 문서처럼 건조하고 설명형으로 맞춘다.
- 대주제 사이에는 `<br />` + `---` 패턴을 공통으로 사용한다.
- 처음 보는 사람도 흐름을 따라 이해할 수 있게 쓴다.
- 실무에서 자주 막히는 문제와 점검 방법을 함께 둔다.

`Portfolio` 문체와 구조 기준은 아래로 본다.

- 기술 위키보다 커리어 문맥이 먼저 보이게 쓴다.
- 프로젝트, 자소서, 면접, 코딩 테스트 문서를 카테고리로 묶는다.
- 기술 참조가 필요하면 `TIL.wiki` 대표 문서로 연결한다.
- 원문 slug는 가능한 한 유지해 추적 가능성을 높인다.
