# [네트워크] REST

## 한 줄 요약

> <u>REST는 “리소스”를 URI로 식별하고, `HTTP method`로 상태를 표현하며, 서버가 세션 상태를 갖지 않는(`stateless`) 제약을 중심으로 하는 API 설계 스타일이다.</u>

REST는 프로토콜이 아니라 설계 원칙에 가깝고, HTTP는 이를 표현하기 좋은 수단이다.

핵심 키워드: `resource`, `URI`, `representation`, `stateless`, `cache`

<br /><br />

---

## 1. 리소스 중심

> <u>행동이 아니라 “대상(명사)”을 먼저 잡는다.</u>

- 예: `/users/42`, `/orders/1001/items`
- 리소스의 표현은 JSON 같은 포맷으로 전달한다.

<br /><br />

---

## 2. HTTP 메서드 의미를 살린다

- `GET`: 조회
- `POST`: 생성(또는 처리)
- `PUT`: 전체 교체
- `PATCH`: 부분 수정
- `DELETE`: 삭제

<br /><br />

---

## 3. Stateless(무상태)

> <u>요청은 독립적이어야 하고, 서버는 클라이언트 세션 상태를 강하게 붙잡지 않는다.</u>

- 필요한 정보는 요청에 포함되어야 한다. (인증/권한 포함)
- 캐시/스케일아웃에 유리해진다.
