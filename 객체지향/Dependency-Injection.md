# Dependency Injection (DI)

> 태그: `#oop` `#design-principle` `#di`<br>
> 작성일: 2026-06-25<br>
> 최종 수정일: 2026-06-25

## 정의

`DI`는 객체가 의존 대상을 직접 생성하지 않고 외부에서 주입받는 방식이며, 결합도를 낮추고 테스트/교체를 쉽게 만든다.

## 특징 / 상세

핵심 키워드: `constructor injection`, `interface`, `IoC container`, `testability`

DI는 "패턴/기법"이고, 스프링 같은 DI 컨테이너는 그걸 도와주는 도구다.

### 1. 왜 쓰나

> 구현을 바꾸기 쉬운 구조를 만들기 위해서다.

- 의존 대상을 교체하기 쉬워진다.
- 단위 테스트에서 mock/stub로 대체하기 쉬워진다.

### 2. 대표 형태

- 생성자 주입: 생성 시점에 의존을 고정한다.
- 세터 주입: 선택적 의존에 쓰이지만, 누락 가능성이 생긴다.

### 3. DIP와의 관계

> DI는 DIP를 구현하는 대표 수단이다.

- 추상(인터페이스)을 기준으로 의존하게 하고, 실제 구현은 밖에서 주입한다.

### 4. 실전 포인트

> DI의 핵심은 "생성 책임을 밖으로 뺀다"는 점이다.

- 직접 `new`를 남발하면 교체와 테스트가 어려워진다.
- 생성자 주입은 의존성이 눈에 보여서 가장 많이 권장된다.
