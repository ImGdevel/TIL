## 정의

`Dynamic Binding`은 호출될 메서드 구현이 런타임에 "실제 객체 타입"으로 결정되는 메커니즘이며, 오버라이딩 기반 다형성의 핵심이다.

## 특징 / 상세

핵심 키워드: `dynamic dispatch`, `virtual`, `vtable`, `runtime`, `polymorphism`

정적 바인딩은 컴파일 시점에 호출 대상이 고정되는 것을 말한다.

### 1. 왜 필요한가

> 상위 타입으로 코드를 작성해도, 실제 구현을 바꿔 끼울 수 있게 한다.

- 인터페이스/추상 타입을 기준으로 의존하면, 구현 변경에 강해진다.

### 2. 정적 vs 동적 바인딩(감각)

- 정적 바인딩: 컴파일 시점에 호출 대상이 결정된다. (오버로딩이 주로 여기에 해당)
- 동적 바인딩: 런타임에 실제 객체 타입으로 결정된다. (오버라이딩)

### 3. 비용과 트레이드오프

> 유연성을 얻는 대신, 간접 호출 비용/디버깅 난이도가 늘 수 있다.

- 대부분의 경우 비용은 작지만, 성능이 극단적으로 민감한 영역에서는 고려 대상이 된다.

### 4. 실전 포인트

> 다형성을 쓰는 이유는 "교체 가능성"을 얻기 위해서다.

- 상위 타입 참조로 코드를 작성하면 구현 변경에 덜 흔들린다.
- 플러그인 구조, 전략 패턴, 인터페이스 기반 설계가 대표적이다.
